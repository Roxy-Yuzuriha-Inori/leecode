# 题目
给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。
请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

- 示例 1：
  - 输入：nums = [100,4,200,1,3,2]
  - 输出：4
  - 解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

# 自解
不能用Array.sort,它的时间复杂度为 O(n log n)
# 原解

```java
class Solution {
    public int longestConsecutive(int[] nums) {
        //将一个数组变为set
        Set<Integer> num_set = new HashSet<Integer>();
        for (int num : nums) {
            num_set.add(num);
        }
        //定义 最长数字连续序列 变量
        int longestStreak = 0;
        //遍历set
        for (int num : num_set) {
            //判断前一个数是否存在于数组，不存在做以下逻辑，存在的不用考虑
            if (!num_set.contains(num - 1)) {
                int currentNum = num;
                int currentStreak = 1;
                //用while循环判断+1之后的数是否还存在于map中
                while (num_set.contains(currentNum + 1)) {
                    //通过当前值+1实现改变while循环条件
                    currentNum += 1;
                    //记录当前数字的最长值
                    currentStreak += 1;
                }
                //不断更新最长值
                longestStreak = Math.max(longestStreak, currentStreak);
            }
        }
        return longestStreak;
    }
}

```
# 知识点
## 时间复杂度排序
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2^n) < O(n!)

## 如何将数组转换为hashset
循环取值 + add方法

# 总结
- 通过判断前一个数是否存在于数组来减少多余的步骤<br>
- while循环的循环条件 可以通过 循环体来实现 可变的循环条件
